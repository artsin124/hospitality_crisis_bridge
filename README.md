🏔️ Himalayan Crest Bridge
Cloud-Native Crisis Management Framework for Alpine Hospitality

Himalayan Crest Bridge is a high-fidelity emergency response system designed to eliminate operational stagnation in remote hospitality nodes, such as Manali. By establishing a Deterministic Telemetry Bridge, we provide guests with real-time visual assurance and AI-guided support during critical incidents.

🚀 Core Features
Context-Aware Ingress: Instant, hardware-agnostic activation via localized QR tokens and URL parameters—no app installation required.

Dynamic Tactical Map: A live schematic overlay providing Spatial Feedback by streaming responder coordinates with sub-second latency.

Command Orchestration: A centralized HQ dashboard for staff to acknowledge alerts, broadcast ETAs, and manage crisis vectors.

Acoustic Emergency Engine: Singleton-pattern siren integration to ensure auditory alerts are clear and synchronized across terminals.

🧠 Google AI & Cloud Integration
This project is built 100% on the Google ecosystem to ensure maximum resilience and scalability:

Gemini API: Acts as the cognitive brain of the bridge, performing Post-Crisis Synthesis to generate professional incident reports and providing real-time, AI-guided first-aid instructions to guests.

Cloud Firestore: Our real-time data fabric, enabling Asynchronous State Synchronization and live telemetry streams.

Firebase Hosting & Auth: Ensures a secure, globally distributed CDN for our Flutter Web frontend.

🛠️ Technical Stack
Frontend: Flutter Web (Hardware-Agnostic UI)

Backend: Firebase (Firestore, Authentication, Hosting)

Language: Dart

AI Service: Gemini API

⚙️ Setup & Installation
Clone the Repository:

Bash

Install Dependencies:

Bash

Configure Firebase:

Create a project in the Firebase Console.

Enable Firestore, Auth, and Hosting.

Run flutterfire configure to sync your local environment.

Deploy to Web:

Bash
flutter build web --release
firebase deploy
🛰️ Architecture Overview
The system utilizes a Spatial Feedback Loop architecture. When a guest triggers an SOS, a Firestore node is created. The Staff HQ acknowledges the signal, establishing a direct telemetry stream where staff X,Y coordinates are mapped onto the guest's tactical schematic in real-time.
